# Phase 4: 차별화 & 성취감

## 차별화 전략 개요

비슷한 할일 앱들(Google Tasks, Todoist 등) 중에서 **이 앱을 계속 쓰게 만드는 핵심 장치**를 구현합니다.

**핵심 인사이트:**
- 직장인(김준호)은 빠른 입력과 반복 자동화가 떠나는 지점
- 학생(박지현)은 마감일 추적과 성취감이 떠나는 지점
- 공통: 간단하지만 강력한 피드백이 필요

---

## 선택된 성취감 & 차별화 기능 (5개 카테고리, 11개 기능)

### 📊 E1. 성취감 & 보상 (2개)

#### E1-1. 완료 체크 시 애니메이션
- **기능:** 할일 체크박스 클릭 → 할일이 회전하며 사라지는 애니메이션
- **효과:** 한 번의 성공을 시각적으로 강화 (도파민 피드백)
- **타겟:** 직장인(김준호): 빠른 업무 처리 후 만족감 강화
- **구현:** Framer Motion 또는 CSS 애니메이션 (0.6초)

#### E1-2. 주간 완료율 차트
- **기능:** 대시보드에 "이번주 완료율 82%" 표시 (원형 차트)
- **효과:** 주간 성취도를 한눈에 파악, 다음주 동기부여
- **타겟:** 학생(박지현): 과제 관리 성공 시각화
- **구현:** Recharts로 간단한 원형 차트 (PC 대시보드)

---

### 🧠 E2. 인지 부담 감소 (3개)

#### E2-1. 오늘 할일 요약
- **기능:** 앱 실행 시 헤더에 "오늘 3개 | 높음 2개"
- **효과:** 한눈에 오늘의 할일량 파악, 정신적 부담 경감
- **타겟:** 직장인(김준호): 아침 출근 후 5초 내 상황 파악
- **구현:** 필터링된 할일 count를 헤더에 표시

#### E2-2. 마감일순 자동 정렬
- **기능:** 기본 정렬 = 마감일순 (지난일 → 오늘 → 내일 → 이번주)
- **효과:** 사용자가 정렬 순서를 고민하지 않음
- **타겟:** 학생(박지현): 마감일이 임박한 과제 자동 우선순위
- **구현:** 마감일로 정렬하는 comparator 함수

#### E2-3. 마감일별 그룹화
- **기능:** 할일 리스트를 "지난일 (2개) | 오늘 (3개) | 내일 (1개) | 이번주 (4개)" 섹션으로 구분
- **효과:** 시간 맥락에 따른 할일 관리 편의성 극대화
- **타겟:** 학생(박지현): 시험 준비 시 주간 할일 한눈에 보기
- **구현:** 할일을 마감일 기준으로 그룹화하는 함수

---

### ✨ E3. 입력 즐거움 & 속도 (2개)

#### E3-1. 빠른 입력 숏컷
- **기능:** `Ctrl+N` 또는 모바일 FAB 클릭 → 팝업창에서 제목만 입력 → Enter로 저장
- **효과:** 회의 중이나 이동 중 3초 안에 할일 추가
- **타겟:** 직장인(김준호): "회의 중 메모앱 속도와 동등"
- **구현:** 모달/드로어 컴포넌트 + 빈 입력값 바로 저장 가능

#### E3-2. 스마트 마감일 입력
- **기능:** 마감일 입력 시 "내일", "금요일", "3주 후" 등 자연어 인식
- **효과:** 날짜 선택의 번거로움 제거
- **타겟:** 직장인(김준호): 빠른 입력 흐름 방해 방지
- **구현:** 파서 함수 (date-fns 또는 chrono 라이브러리 사용)

---

### 📈 E4. 변화 & 맥락 (1개)

#### E4-1. 이번주 vs 지난주 비교
- **기능:** 대시보드에 "지난주 12개 완료 vs 이번주 8개 (현재까지)"
- **효과:** 주간 생산성 트렌드를 직관적으로 파악, 동기부여
- **타겟:** 직장인 & 학생: "이번주는 좀 덜 바쁜가?" 객관적 판단
- **구현:** localStorage 히스토리에서 지난주 완료 수를 계산

---

### 🔔 E5. 부드러운 강제력 (2개)

#### E5-1. 마감일 알림
- **기능:** 마감일 전날 저녁(6시) + 당일 아침(8시) 브라우저 알림
- **효과:** 마감일을 놓치지 않도록 상기
- **타겟:** 학생(박지현): 과제 마감일 미리 숙지
- **구현:** Web Notifications API + 로컬 타이머

#### E5-2. 지난 마감일 강조
- **기능:** 마감일이 지난 할일을 빨강으로 표시 + 경고 아이콘(⚠️) 추가
- **효과:** "아, 이 과제 마감일 지났네?" 시각적 상기
- **타겟:** 학생(박지현): 과제 연체 방지
- **구현:** 마감일 < 오늘 조건으로 스타일 적용

## 구현 우선순위

### MVP (M1~M4에 포함, 필수)
1. **E2-2 (마감일순 정렬)** — 기본 기능, 모든 시나리오의 기초
2. **E2-3 (마감일별 그룹화)** — 학생 타겟, 시각적 가치 높음
3. **E3-1 (빠른 입력 숏컷)** — 직장인 타겟, 사용성 핵심
4. **E5-2 (지난 마감일 강조)** — 학생 타겟, 마감일 놓치기 방지

### M5 이후 (선택, 성취감 강화)
5. **E1-1 (완료 애니메이션)** — 재미, 게이미피케이션
6. **E2-1 (오늘 요약)** — 직장인 아침 습관 형성
7. **E5-1 (마감일 알림)** — 알림 시스템 필요

### v2로 미루기 (고급, 데이터 축적 필요)
8. **E1-2 (완료율 차트)** — 통계, 데이터 필요
9. **E3-2 (스마트 마감일)** — 자연어 처리, 복잡
10. **E4-1 (주간 비교)** — 히스토리 데이터 필요

## 의도적으로 넣지 않는 것

| 기능 | 이유 |
|------|------|
| **알림 푸시** | 사용자가 앱을 매일 열지 않으므로 ※ v2에서 고려 |
| **음성 입력** | 모바일에서 음성인식 라이브러리의 복잡성 |
| **할일 템플릿** | 직장인은 보통 템플릿을 안 씀 (개인차 큼) |
| **국제화** | 한국 사용자만 대상 (시간 예산 절감) |
| **협업 공유** | 개인용 앱이므로 제외 |
| **할일 이미지** | 할일은 텍스트 기반 (이미지 필요 없음) |

## 핵심 차별화 포인트

### 직장인(김준호)을 붙잡는 방법
1. **E3-1 (빠른 입력):** "30초 안에 끝나네" → 습관화
2. **E2-2 (마감일순):** "내가 할 일 순서가 자동 정렬된다" → 신뢰
3. **E1-1 (애니메이션):** "할일 완료할 때 기분 좋다" → 중독성

### 학생(박지현)을 붙잡는 방법
1. **E2-3 (그룹화):** "이번주 과제들이 한눈에 보인다" → 안심
2. **E5-2 (지난 마감):** "마감 지난 과제를 못 놓친다" → 신뢰
3. **E2-1 (오늘 요약):** "오늘 뭐 해야 하는지 바로 알 수 있다" → 효율성

- **직장인:** 월 50회 이상 사용 (주간 10회+)
- **학생:** 시험 기간 일일 3회 이상 방문
- **전체:** 반복 방문율 60% 이상 (1주일 내)
