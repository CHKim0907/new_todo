# Phase 5: 디자인

## 확정 디자인: 시안 B (섹션 카드)

### 디자인 철학
**명랑하고 친근한 할일 앱**
- 마감일별 섹션 그룹화 (지난일→오늘→내일→이번주)
- 세로 스크롤 중심, 모바일 최적화
- 섹션별 따뜻한 색상으로 시각적 명확성
- 카드 스타일로 각 할일을 독립적인 작업으로 표현

```

요소:

```

│   📝 [설명] (선택)           │

│   🔄 [반복]                  │

배경: #FFFFFF
테두리: 1px #E8E8E8
모서리: 8px border-radius
패딩: 16px

그림자: 0 1px 4px rgba(0,0,0,0.06)
호버: 0 2px 8px rgba(0,0,0,0.12)

요소 상세:
├── 체크박스: 24px × 24px, 둥근 모서리
│   └── 완료 시 애니메이션 (회전 + 사라짐, 0.6초)
├── 제목: 16px / 600 weight / #333333
│   └── 완료 시: 취소선 + #999999

│   └── 오늘: #FF7AAC (분홍)
│   └── 내일+: #999999 (회색)
├── 우선순위 배지: 12px 폰트, 둥근 배경

## 인터랙션 (Interactions)

### I1. 할일 완료 (Complete)
```
사용자 액션: 체크박스 클릭
애니메이션:
1. 체크박스 회전 (0.3초, cubic-bezier)
2. 할일 카드 페이드 아웃 (0.3초)
3. 카운트 업데이트 (즉시)
피드백: "완료!" 토스트 메시지 (2초 후 자동 닫힘)

복구: "실행 취소" 버튼 (토스트에 내장)
```

### I2. 할일 추가 (Create)
```
사용자 액션: FAB 클릭 또는 Ctrl+N
모달 열기: 하단 드로어 (모바일) / 중앙 모달 (PC)
포커스: 제목 입력 필드 (자동)
Enter 키: 저장 후 모달 닫힘 + 확인 메시지
Esc 키: 모달 닫힘 (입력 내용 버림)

최소화: 제목만 입력 → 나중에 상세 입력
```

### I3. 할일 수정 (Update)
```
사용자 액션: 카드 클릭 또는 메뉴 [수정]
화면 전환: 상세 입력 페이지
입력 필드: 제목, 설명, 마감일, 우선순위, 반복
저장: 자동 저장 또는 [저장] 버튼
뒤로: 목록으로 돌아가기
```

### I4. 할일 삭제 (Delete)
```
사용자 액션: 메뉴 [삭제] 클릭
확인: "정말 삭제할까요?" 다이얼로그
선택지: [삭제] [취소]
피드백: "삭제되었습니다" 토스트
복구: 실행 취소 (5초)
```

### I5. 필터링 (Filter)
```
사용자 액션: 필터 버튼 클릭
드롭다운: 옵션 표시 (자동 닫힘 또는 유지)
적용: 실시간 업데이트 (즉시)
시각적 피드백: 활성 필터 배경 강조
```

### I6. 마감일 입력 (Due Date)
```
사용자 액션: 마감일 필드 클릭
입력 방식: 달력 선택 (모바일 date picker / PC 캘린더)
스마트 입력: "내일", "금요일" 등 자연어 (v2)
시간 설정: 선택 (기본: 23:59)
확인: 날짜 표시로 시각화
```

### I7. 반복 설정 (Repeat)
```
사용자 액션: 반복 필드 클릭
옵션: 없음 / 매일 / 주간 / 월간
종료 날짜: 선택 (선택사항)
자동 생성: 설정 후 다음 날자동 생성
시각적 표시: 🔄 아이콘 + "매주"
```

## 접근성 (Accessibility)

### A1. 키보드 네비게이션
```
Tab: 포커스 이동 (체크박스 → 제목 → 메뉴)
Enter: 체크박스 토글 또는 버튼 활성화
Space: 버튼 활성화
Delete: 할일 삭제 (포커스 시)
Escape: 모달/메뉴 닫기
Ctrl+N: 빠른 추가
```

### A2. 포커스 표시
```
포커스 링: 2px 파란색 테두리 (#4D96FF)
모서리: 대상 요소의 border-radius + 2px
간격: 2px 여백
호버: 배경 톤 조정으로도 피드백
```

### A3. 색상 외 정보 전달
```
상태 표시:
- 완료: 취소선 + 회색 텍스트 + 체크 아이콘
- 마감 지남: 빨강 + ⚠️ 아이콘
- 높은 우선순위: 빨강 + ⭐⭐ 배지

아이콘:
- 체크: ☐ / ☑
- 삭제: 🗑️ 또는 X 버튼
- 수정: ✏️ 또는 [수정] 텍스트
- 반복: 🔄
- 마감: ⏰
```

### A4. 명도 대비 (Contrast)
```
텍스트 vs 배경: WCAG AA 4.5:1 이상
- #333333 텍스트 on #FFFFFF: 12.6:1 ✓
- #666666 텍스트 on #FFFFFF: 7.5:1 ✓
- #FFFFFF 텍스트 on #FF7AAC: 5.2:1 ✓

인터랙티브 요소: WCAG AA 3:1 이상
```

### A5. 스크린 리더 지원
```
ARIA 라벨:
- 체크박스: aria-label="{{title}} 완료 여부"
- 버튼: aria-label="{{action}} 버튼"
- 섹션: role="region" aria-label="{{section name}}"

역할:
- 섹션: <section role="region">
- 버튼: <button aria-pressed="...">
- 목록: <ul role="list">
```

## 애니메이션 사양 (Animation Specs)

### 완료 할일 사라지기
```
Duration: 0.6초
Easing: cubic-bezier(0.34, 1.56, 0.64, 1) - 탄성
Sequence:
1. 체크박스 회전: 0° → 360°, 0.3초
2. 카드 스케일 + 페이드: 1 → 0.8, opacity 1 → 0, 0.3초
3. 높이 축소: auto → 0, 0.2초
```

### 섹션 열기/닫기
```
Duration: 0.3초
Easing: cubic-bezier(0.4, 0, 0.2, 1)
효과: 높이 애니메이션 (auto height 계산)
```

### 호버 효과
```
Duration: 0.2초
Easing: ease-out
효과: 그림자 강화, 배경 톤 밝아짐
```
