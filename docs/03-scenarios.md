
# Phase 3: 시나리오

## 시나리오 개요

두 페르소나의 실제 사용 상황을 시간대별·상황별로 구체화합니다.
각 시나리오는 **Context → Flow → 설계 인사이트**로 구성됩니다.

---

## 직장인(김준호) 시나리오

### S1. 아침 출근 후 하루 계획 세우기

**Context**
- 시간: 오전 9시 (출근 직후)
- 디바이스: PC (책상에 앉음)
- 상황: 어제 완료하지 못한 일과 오늘 해야 할 일 확인

**Flow**
1. 앱 실행
2. 오늘 마감일인 할일들을 최상단에 확인
3. 어제 안 한 일들 중 우선순위 높은 것 먼저 보기
4. 새로운 할일 3개 추가 (마감일: 오늘, 내일, 금일)
5. "오늘은 이것들을 꼭 끝내자" 다짐

**설계 인사이트**
- 마감일이 오늘인 할일을 최상단에 배치 필수
- 우선순위순 정렬 기능 필수
- PC에서 여러 할일을 빠르게 추가할 수 있는 UX 필요

---

### S2. 회의 중 할일 발생 (모바일)

**Context**
- 시간: 오전 10시 (팀 미팅 중)
- 디바이스: 모바일 (스마트폰)
- 상황: 회의 중 새로운 태스크 할당받음

**Flow**
1. 회의는 계속 진행 중
2. 휴대폰에서 메모앱을 대신 앱 실행
3. "신입 온보딩 자료 준비" 입력
4. 나중에 마감일, 우선순위 추가할 생각
5. 회의 끝나고 편한 마음으로 정리

**설계 인사이트**
- 모바일에서 제목만 빠르게 입력 가능해야 함 (1~2초)
- 다른 정보(마감일, 우선순위)는 나중에 추가 가능하도록
- 모바일 입력 필드가 크고 터치하기 편해야 함 (44px 이상)

---

### S3. 업무 중 우선순위 재조정

**Context**
- 시간: 오후 2시 (업무 중)
- 디바이스: 모바일
- 상황: 긴급 할일이 생겨서 우선순위 재조정 필요

**Flow**
1. 앱 실행
2. "마감일 오늘" 필터로 보기
3. 새 긴급 할일 추가 (우선순위: 높음)
4. 기존 할일들의 우선순위 재조정
5. "높음" 필터로 다시 보기
6. 업무로 돌아가기

**설계 인사이트**
- 우선순위순 기본 정렬 필요
- 필터링 기능 필수 (마감일, 우선순위별)
- 할일 순서를 드래그로 재정렬할 수 있으면 좋음 (v2)

---

### S4. 매주 반복되는 회의/보고 자동화

**Context**
- 시간: 금요일 오후 5시
- 디바이스: PC
- 상황: 매주 반복되는 "주간 보고서 작성" 자동으로 생성되기를 원함

**Flow**
1. 앱 실행 (금요일)
2. "주간 보고서 작성" 할일 생성
3. 반복 설정: 매주 (매주 금요일 자동 생성)
4. 마감일: 매주 금요일 오후 6시 (알림 설정)
5. 다음주 금요일에 자동으로 할일 생성됨
6. 김준호는 반복 설정 후 매번 추가할 필요 없음

**설계 인사이트**
- 반복 설정 UI는 단순해야 함 (매일/주간/월간만)
- 반복 할일은 자동으로 다음 날짜에 생성되어야 함
- 마감일과 함께 반복 기능이 통합되어야 함

## 학생(박지현) 시나리오

### S5. 강의 후 과제 받으면 즉시 기록

**Context**
- 시간: 오후 3시 (강의 후)
- 디바이스: 모바일 (캠퍼스 이동 중)
- 상황: 교수가 과제 할당, 마감일 확인 후 입력

**Flow**
1. 모바일로 앱 실행
2. "프로그래밍 과제 - 정렬 알고리즘 구현" 입력
3. 마감일: 3월 20일 (정확한 날짜 입력, 중요!)
4. 우선순위: 높음 (4개 과제 중 가장 어려움)
5. 설명 추가: "시간복잡도 O(n log n) 이상"

**설계 인사이트**
- 마감일 입력이 명확하고 쉬워야 함 (달력 선택 또는 날짜 입력)
- 마감일이 임박한 할일을 시각적으로 강조 필수
- 설명 필드는 선택이지만, 있으면 좋음

---

### S6. 주간 계획 세우기

**Context**
- 시간: 일요일 저녁 9시
- 디바이스: PC (책상에서)
- 상황: 새로운 주간 과제 계획 수립

**Flow**
1. PC에서 앱 실행
2. 지난주 완료한 과제들을 보며 뿌듯함 느끼기
3. 이번주 할 과제들을 마감일순으로 입력
4. "수요일 데이터베이스 리포트" (3월 19일)
5. "금요일 웹 프로젝트 제출" (3월 21일)
6. "토요일 알고리즘 문제 풀이" (3월 22일)
7. 마감일순으로 정렬되어 표시됨

**설계 인사이트**
- 마감일순 정렬 기본 제공
- 완료한 할일 기록 유지 필요 (삭제하지 않기)
- 이번주 vs 다음주 구분 표시 필수
- 한눈에 이번주 마감일들이 보여야 함

---

### S7. 시험 준비 (중장기 계획)

**Context**
- 시간: 시험 2주일 전
- 디바이스: PC
- 상황: 시험 과목별로 복습 계획 수립

**Flow**
1. 앱 실행
2. "데이터베이스 - Chapter 1~3 복습" (3월 26일)
3. "운영체제 - 프로세스 정리" (3월 27일)
4. "알고리즘 - 동적 프로그래밍 풀이" (3월 28일)
5. 모두 우선순위 높음으로 설정
6. 마감일순으로 정렬되어 보기 좋음
7. 완료할 때마다 체크 (성취감)

**설계 인사이트**
- 마감일 기반 시각화가 핵심
- 마감일이 지난 할일 강조 표시 (빨강 또는 아이콘)
- 완료 체크 후에도 기록으로 남아야 함 (나중에 보기)

---

### S8. 과제 제출 전 최종 확인

**Context**
- 시간: 금요일 오후 (제출 1시간 전)
- 디바이스: PC & 모바일 병행
- 상황: 마감일이 오늘인 과제들 최종 점검

**Flow**
1. PC에서 앱 실행
2. "마감일 오늘" 필터로 보기
3. 3개의 과제가 모두 준비 상태 확인
4. 모바일로 전환 (학교 이동 중)
5. 마감일이 오늘인 할일들 한번 더 확인
6. 제출 직전 마지막 체크

**설계 인사이트**
- PC와 모바일에서 동기화되어야 함
- 마감일 필터는 필수 기능
- 오늘 마감인 할일을 상단 강조 필수

## 설계 원칙 (Design Principles)

| 번호 | 원칙 | 영향받는 시나리오 | 설계 반영 |
|------|------|------------------|---------|
| P1 | 마감일 중심 시각화 | S5, S6, S7, S8 | 마감일순 기본 정렬 + 오늘 할일 최상단 |
| P2 | 모바일-우선 입력 | S1, S2, S3 | 제목만으로 30초 이내 입력 가능 |
| P3 | 우선순위 기반 필터링 | S3, S4 | 우선순위별 필터 + 필터된 리스트 정렬 |
| P4 | 반복 자동화 | S4 | 매일/주간/월간 반복 설정 후 자동 생성 |
| P5 | 완료 기록 유지 | S6, S7 | 완료 체크 후에도 삭제하지 않음 (숨김 또는 별도 탭) |
| P6 | PC/모바일 동기화 | S8 | localStorage 자동 동기화 |
| P7 | 마감일 경고 시각화 | S5, S7 | 지난 마감일: 빨강 / 오늘: 주황 / 내일: 노랑 |
| P8 | 빠른 추가 + 나중 편집 | S2 | 제목 입력 후 나중에 마감일/우선순위 추가 가능 |

## 핵심 인사이트 요약

### 김준호 (직장인)
- **구매 이유:** "모바일에서 빠르고, 반복은 자동으로"
- **성공 조건:** 마감일 오늘 할일 상단 + 모바일 30초 입력
- **포기 지점:** 입력이 3초 이상 걸리면 메모앱으로 돌아감

### 박지현 (학생)
- **구매 이유:** "마감일 명확하고, 과제들이 한눈에 보임"
- **성공 조건:** 마감일순 정렬 + 마감일 경고 표시
- **포기 지점:** 마감일 입력이 복잡하면 종이 일정표로 돌아감

### 공통 설계 원칙
→ **마감일 중심, 모바일 우선, 빠른 입력**
