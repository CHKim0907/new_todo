# Phase 1: 요구사항

## 프로젝트 개요

**앱 이름:** 할일 관리 앱  
**목적:** 개인용 할일 관리 및 추적  
**주 사용자:** 개인 (1인)  
**사용 디바이스:** 반응형 (PC 및 모바일)  
**기술 스택:** React + TypeScript  
**데이터 저장:** localStorage (브라우저 로컬 저장)  
**시간 예산:** 하루 (6~8시간)

---

## 핵심 기능 요구사항

### 데이터 모델

```
할일 (Todo)
├── id: string (UUID)
├── title: string (할일 제목)
├── description?: string (상세 설명)
├── priority: 'high' | 'medium' | 'low' (우선순위)
├── completed: boolean (완료 여부)
├── dueDate?: Date (마감일)
├── repeat?: {
│   ├── type: 'daily' | 'weekly' | 'monthly' | null
│   └── endDate?: Date
├── createdAt: Date (생성일)
└── updatedAt: Date (수정일)
```

### 기능 명세

#### F1. 할일 생성 (Create)
- 제목 입력 (필수)
- 우선순위 선택 (필수, 기본값: 중간)
- 설명 입력 (선택)
- 마감일 설정 (선택)
- 반복 설정 (선택: 없음/매일/주간/월간)

#### F2. 할일 조회 (Read)
- 모든 할일을 리스트로 표시
- 우선순위 순(높→중→낮) 또는 입력 순서로 정렬
- 완료/미완료 상태 시각적 표시 (체크박스)
- 마감일이 지난 할일 경고 표시

#### F3. 할일 수정 (Update)
- 제목, 설명, 우선순위, 마감일, 반복 수정
- 완료 상태 토글 (체크박스 클릭)

#### F4. 할일 삭제 (Delete)
- 할일 삭제 (확인 다이얼로그)
- 완료된 할일 일괄 삭제

#### F5. 필터링 & 정렬
- 우선순위별 필터링 (높/중/낮/모두)
- 상태별 필터링 (완료/미완료/모두)
- 마감일순 정렬
- 우선순위순 정렬

#### F6. 반복 할일 자동 생성
- 마감일이 지나면 다음 반복 날짜로 새 할일 자동 생성
- 반복 종료일까지만 자동 생성

#### F7. 데이터 영속성
- localStorage에 자동 저장
- 새로고침 후에도 데이터 유지
- 내보내기/가져오기 (JSON)

## 비기능 요구사항

### 성능
- 할일 100개 이상에서도 1초 이내 렌더링
- localStorage 쓰기/읽기 지연 최소화

### 접근성
- 키보드 네비게이션 가능 (Tab, Enter, Delete)
- 체크박스, 버튼 등 포커스 표시 명확
- 색상 외 수단으로도 정보 전달 (아이콘, 텍스트)

### 반응형
- PC (1024px 이상): 여유로운 레이아웃
- 태블릿 (768px~1023px): 단일 컬럼
- 모바일 (767px 이하): 터치 친화적 (버튼 >= 44px)

### UX
- 명확한 상태 피드백 (생성/수정/삭제 알림)
- 실행 취소 기능 (선택)
