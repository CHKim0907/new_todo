# Phase 2: 페르소나

## 페르소나 개요

이 할일 앱은 **두 가지 사용자 타입**을 지원합니다:
- **주 페르소나:** 직장인 (빠른 입력, 간단한 관리)
- **대조 페르소나:** 학생 (마감일 중심, 상세 추적)

---

## 주 페르소나: 직장인 김준호

### 프로필
- **이름:** 김준호
- **나이:** 32세
- **직업:** IT 회사 개발팀 팀장
- **기술 수준:** 높음 (개발자)
- **주요 특징:** 바쁜 업무, 회의 많음, 스마트폰 의존도 높음

### 목표
1. 업무 중에 빠르게 할일을 추가하고 추적하기
2. 반복되는 일주일 루틴을 자동화하기
3. 모바일에서 신속하게 입력하고 확인하기
4. 최소한의 클릭으로 완료 표시하기

### 현재 불만 (Pain Points)
- Notion은 너무 복잡해서 입력이 오래 걸림
- Google Tasks는 반복 설정이 불편함
- 휴대폰에서 빠르게 추가하려면 메모 앱을 써야 함
- 매주 반복되는 회의나 보고서를 매번 손으로 입력해야 함
- 우선순위가 모호할 때 결정이 어려움

### 행동 패턴
- **시간대:** 아침 회의 전(9시), 점심(12시), 퇴근 전(6시) 3회 확인
- **입력 방식:** 모바일에서 수시로 추가 (버스, 회의 대기 시간 등)
- **할일 특성:** 
  - 짧은 할일 (5줄 이하 텍스트)
  - 우선순위는 고 / 중 / 저 3단계만 필요
  - 마감일은 대부분 내일 또는 이번주
- **주간 루틴 예시:**
  - (매주 월) 주간 계획 수립
  - (매주 수) 팀 미팅 준비
  - (매주 금) 주간 보고서 작성

### 성공 기준
- "30초 이내에 할일을 추가할 수 있다"
- "반복 할일은 설정 후 자동으로 생성된다"
- "휴대폰에서 불편함 없이 사용할 수 있다"

## 대조 페르소나: 학생 박지현

### 프로필
- **이름:** 박지현
- **나이:** 21세
- **신분:** 대학생 (컴퓨터공학과 3학년)
- **기술 수준:** 중상 (일반 사용자)
- **주요 특징:** 과제 많음, 시험 기간 변수 큼, 계획적 학습 필요

### 목표
1. 과제별 마감일을 명확히 추적하기
2. 우선순위를 통해 공부 계획 세우기
3. 이번주, 이번달 할일을 한눈에 보기
4. 완료한 과제들을 기록으로 남기기

### 현재 불만 (Pain Points)
- 과제 마감일을 놓치는 경우가 있음
- 여러 앱에 흩어져 있는 할일들을 한곳에서 보고 싶음
- 시험 기간에 공부 계획을 체계적으로 세우기 어려움
- 완료한 과제들의 기록을 남길 곳이 없음
- 동일한 시간의 과제들이 쌓일 때 우선순위 결정이 어려움

### 행동 패턴
- **시간대:** 아침(등교 전), 저녁(공부 전), 밤(취침 전) 확인
- **입력 방식:** 컴퓨터에서 주간 계획, 모바일에서 수시 추가
- **할일 특성:**
  - 과제명 + 과목 + 마감일까지 기록
  - 우선순위: 마감일이 임박한 것부터
  - 반복 할일: 거의 없음 (학기별로 달라짐)
- **학기 중 유형:**
  - 정기적 과제 (매주 프로그래밍 과제)
  - 비정기적 과제 (레포트, 프로젝트)
  - 시험 준비 (주간/월간 복습)

### 성공 기준
- "마감일이 지난 과제를 시각적으로 강조한다"
- "이번주 할일과 다음주 할일을 구분해서 볼 수 있다"
- "완료한 과제 기록을 볼 수 있다"

## 두 페르소나의 긴장 축 비교

| 차원 | 직장인 (김준호) | 학생 (박지현) | 설계 영향 |
|------|---------------|------------|---------|
| **입력 속도** | 빠름 (30초 목표) | 정확함 (완전한 정보) | 최소 정보로 빠른 입력 + 나중에 수정 가능 |
| **정보량** | 간단 (제목만) | 상세 (제목+설명) | 제목 필수, 설명/마감일 선택 |
| **마감일 중요도** | 중간 (내일/이번주) | 높음 (정확한 날짜) | 마감일 시각화 + 경고 기능 필수 |
| **반복 할일** | 높음 (주간 루틴) | 낮음 (학기별 변함) | 반복 설정 UI 단순화 |
| **우선순위** | 간단 (고/중/저) | 복잡 (마감일 기반) | 고/중/저 + 자동 정렬 옵션 |
| **디바이스** | 모바일 우선 | PC/모바일 병행 | 반응형 필수 + 모바일 UX 최적화 |
| **정렬 선호** | 우선순위순 | 마감일순 | 정렬 방식 선택 가능 |

## 설계 영향 & 우선순위

### 긴장 해소 방안

1. **입력 속도 vs 정보 완성도**
   - 해결: 제목만 입력 가능 → 필요시 마감일/우선순위 추가
   - 빠른 입력을 기본, 상세 입력을 선택으로

2. **모바일 우선 vs PC 상세 작업**
   - 해결: 반응형 디자인 + 모바일에서도 모든 기능 접근 가능
   - 모바일: 터치 친화적 (버튼 크기 >= 44px)
   - PC: 여유로운 레이아웃

3. **반복 설정의 복잡도**
   - 해결: 직장인: 매일/주간/월간만 지원
   - 학생: 필요시 설명에 기록 (앱은 단순 유지)

4. **마감일 가시성**
   - 해결: 오늘/내일/이번주/지난 할일 구분 표시
   - 마감일 지난 항목 강조 (빨간색/아이콘)

## 핵심 인사이트

- **김준호의 성공:** 30초 이내 입력 + 자동 반복
- **박지현의 성공:** 마감일 명확화 + 우선순위 자동 정렬
- **공통 성공:** 반응형 설계로 PC/모바일 모두 최적화

→ **설계 원칙:** 
1. 모바일-우선 입력 경험 (김준호)
2. 마감일 기반 시각화 (박지현)
3. 간단하지만 확장 가능한 구조
